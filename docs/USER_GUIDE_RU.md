# Руководство пользователя rsmsg

rsmsg — настольный мессенджер с шифрованием сообщений и файлов. Приложение работает через сервер rsmsg, но сервер не получает открытый текст сообщений, файлов и имён файлов.

## Что нужно для входа

- Адрес сервера. Для основного приватного сервера: `https://rsmsg.kevindev64.ru`.
- Никнейм.
- Пароль длиной минимум 6 символов.
- Одноразовый invite-код для регистрации нового аккаунта.

Invite-код нужен только для создания аккаунта. После регистрации вход выполняется по никнейму и паролю.

## Первый запуск

1. Откройте приложение rsmsg.
2. Проверьте поле сервера.
3. Если сервер не указан, введите `https://rsmsg.kevindev64.ru` или адрес своего сервера.
4. Введите никнейм и пароль.
5. Если аккаунт уже создан, нажмите `Login`.
6. Если аккаунта ещё нет, нажмите `Create account` и введите invite-код.

Если сервер требует более новую версию приложения, rsmsg покажет сообщение об обновлении.

## Создание аккаунта

1. Нажмите `Create account`.
2. Введите никнейм.
3. Введите пароль.
4. Введите invite-код.
5. Нажмите кнопку создания аккаунта.

Invite-код одноразовый. Если код уже использован или истёк, попросите новый код у администратора сервера.

## Настройки сервера

Адрес сервера можно изменить в `Settings -> Connection`.

Обычный пользователь должен вводить адрес без внутреннего порта. Например:

```text
https://rsmsg.kevindev64.ru
```

Не используйте адрес вида `https://rsmsg.kevindev64.ru:4222`, если сервер стоит за nginx или другим reverse proxy.

## Чаты

Чтобы начать чат:

1. Войдите в аккаунт.
2. В поле `Peer nickname` введите никнейм собеседника.
3. Нажмите `Search users`, если хотите проверить наличие пользователя.
4. Нажмите `Open chat`.
5. После открытия чата напишите сообщение и нажмите `Send`.

Если сообщения не приходят сразу, нажмите `Sync incoming`. Обычно синхронизация выполняется автоматически.

## Сообщения

Сообщения шифруются на устройстве отправителя и расшифровываются на устройстве получателя. Сервер хранит только зашифрованные данные.

Статусы сообщений:

- `Sending`: сообщение отправляется.
- `Sent`: сервер принял сообщение.
- `Delivered`: получатель получил сообщение с сервера.
- `Read`: получатель открыл чат и сообщение отмечено прочитанным.
- `Failed`: отправка не удалась.

Если приложение показывает, что получены pending messages, но они не расшифрованы, переоткройте чат с собеседником. Если приложение показывает предупреждение о смене ключа, используйте `Trust new key` только если вы уверены, что это тот же собеседник.

## Файлы

Можно отправлять файлы размером до `100 MB`.

Файлы шифруются на устройстве отправителя. Сервер не видит содержимое файла, имя файла и ключ файла в открытом виде.

Чтобы отправить файл:

1. Откройте чат.
2. Нажмите `Attach file`.
3. Выберите файл.
4. Дождитесь окончания загрузки.

Чтобы сохранить входящий файл, нажмите `Save` рядом с сообщением-файлом.

## Аудио- и видеозвонки

Звонить можно только пользователю, который сейчас online.

Звонки используют WebRTC. Медиа шифруется стандартным WebRTC DTLS-SRTP. Это защищает аудио и видео в канале связи. Это отличается от шифрования сообщений и файлов, где rsmsg использует собственный application-level E2EE поверх серверного хранения.

TURN-сервер, если он используется, ретранслирует зашифрованные SRTP-пакеты и не должен видеть открытые аудио/видео данные.

В окне звонка доступны:

- принять звонок;
- отклонить звонок;
- отключить микрофон;
- включить микрофон;
- выключить камеру;
- включить камеру;
- завершить звонок.

Горячие клавиши:

- `Esc`: завершить звонок.
- `Ctrl+M` или `Command+M`: включить или выключить микрофон.

## Настройки звонков

Устройства можно выбрать в `Settings -> Call devices`:

- микрофон;
- динамик;
- камера.

Сетевые настройки звонков находятся в `Settings -> Call network`.

По умолчанию используется публичный STUN-сервер:

```text
stun:stun.l.google.com:19302
```

Для стабильных звонков через NAT или строгий firewall может понадобиться TURN. Данные TURN выдаёт администратор сервера.

## Блокировка пользователей

Пользователя можно заблокировать в чате или в `Settings -> Privacy`.

После блокировки:

- заблокированный пользователь не сможет отправлять вам сообщения;
- вы не сможете отправлять сообщения заблокированному пользователю;
- звонки между вами будут запрещены.

Разблокировать пользователя можно в `Settings -> Privacy`.

## Локальные данные

rsmsg хранит локальные ключи, сессии и историю на вашем устройстве.

Стандартные директории:

- macOS: `~/Library/Application Support/rsmsg`
- Windows: `%APPDATA%\rsmsg`
- Linux: `$XDG_DATA_HOME/rsmsg` или `~/.local/share/rsmsg`

Для тестирования нескольких аккаунтов на одной машине можно использовать разные профили:

```bash
RSMSG_PROFILE=alice rsmsg
RSMSG_PROFILE=bob rsmsg
```

Если удалить локальные данные, приложение потеряет локальную историю, ключи устройства и сохранённые сессии.

## Обновления

Приложение проверяет manifest обновлений по адресу:

```text
https://kevindev64.ru/rsmsg-downloads/stable/manifest.json
```

Если обновление обязательное, вход может быть заблокирован до установки новой версии.

## Частые проблемы

### Неверный логин или пароль

Проверьте никнейм и пароль. Если пароль утерян, восстановление пароля в текущей версии не реализовано.

### Сервер недоступен

Проверьте адрес сервера в `Settings -> Connection`. Для приватного сервера используйте `https://rsmsg.kevindev64.ru`.

### Пользователь offline

Звонок возможен только если получатель online. Попросите получателя открыть приложение и войти в аккаунт.

### Сообщения не расшифровываются

Переоткройте чат с собеседником. Если появилось предупреждение о смене ключа, подтвердите новый ключ только после проверки, что это действительно устройство собеседника.

### Звонок не соединяется

Проверьте микрофон, камеру и сетевые настройки. Если оба пользователя находятся за NAT или firewall, нужен TURN-сервер.
